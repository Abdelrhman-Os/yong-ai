# Agent Instructions

## Project Overview

Yong AI is a full-stack web application.

## Agent Skills

This project uses Matt Pocock's engineering skills. The following skills are available:

- **grill-me** — Relentless interview about a plan/design
- **grill-with-docs** — Interview about a plan + update CONTEXT.md & ADRs
- **to-prd** — Turn conversation context into a PRD (GitHub issue)
- **to-issues** — Break a plan into grabbable GitHub issues
- **tdd** — Test-driven development with red-green-refactor
- **diagnose** — Disciplined bug diagnosis loop
- **triage** — Triage issues through a state machine
- **improve-codebase-architecture** — Find refactoring opportunities
- **zoom-out** — Get broader context on unfamiliar code
- **prototype** — Build throwaway prototypes
- **handoff** — Compact conversation into a handoff document
- **caveman** — Ultra-compressed communication mode

## Workflow

1. Use **grill-with-docs** to deeply plan a feature and document decisions
2. Use **to-prd** to create a Product Requirements Document
3. Use **to-issues** to break the PRD into individual GitHub issues
4. Use **triage** to manage and prioritize issues
5. Use **tdd** to build features with test-first development
6. Use **diagnose** when debugging issues
7. Use **handoff** when ending a session to preserve context
